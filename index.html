What is Cryptography?

Cryptography: Cryptography is a method of protecting information and communications using codes, so that only those for whom the information is intended can read and process it. In computer science, cryptography refers to secure information and communication techniques derived from mathematical concepts and a set of rule-based calculations called algorithms, to transform messages in ways that are hard to decipher. These deterministic algorithms are used for cryptographic key generation, digital signing, verification to protect data privacy, web browsing on the internet and confidential communications such as credit card transactions and email.

What isClassical Cryptography?

Cryptography: Cryptography strategies and methods used to convert plain text into cipher text without the use of technology are known as classical cryptography. Cryptography is commonly defined as ‚Äúan art of writing hidden messages‚Äù‚Äîand classical cryptography was indeed an art.

Cryptography techniques

Cryptography techniques:Cryptographic techniques are methods used to ensure the secrecy and integrity of data by preventing unauthorized access and modification. These techniques rely on mathematical algorithms and keys to transform information into an unreadable format (ciphertext) and back again (plaintext). They are crucial for secure communication, data storage, and digital signatures.

Plaintext: refers to the original, readable content of data that needs to be transmitted over an insecure network before encryption is applied.

Encrypting Key: is a randomly generated string of bits used to transform plaintext into unreadable ciphertext using mathematical algorithms. There are two main types of encryption keys: symmetric keys and asymmetric keys.

Ciphertext: is the encrypted output produced after applying the encryption algorithm to plaintext. It is not understandable by humans without proper decryption.

Decrypting Key: is used to convert ciphertext back into its original plaintext form. Depending on the encryption method used, this key may be symmetric (the same as the encrypting key) or asymmetric (a different, mathematically related key).

Functionality of a Cryptosystem

Authentication: Authentication is the process of verifying the identity of a user or device. This typically involves the use of a username and password or other credentials. It ensures that only legitimate users are granted access to a network or system.

Authorization: Authorization is the process of determining whether an authenticated user has permission to access specific resources or perform certain actions. Most modern security systems follow a two-step approach:

  1. Step 1: The user is authenticated (identity is verified).
  2. Step 2: The system determines what resources or services the authenticated user is allowed to access.

Confidentiality (Privacy): Confidentiality ensures that sensitive information is accessible only to authorized individuals. Through the use of encryption, plaintext data is transformed into ciphertext, making it unreadable to anyone who does not possess the correct decryption key. This process protects the privacy of the data during storage or transmission.

Integrity: Integrity guarantees that the data has not been altered, tampered with, or corrupted during transmission. It ensures that the received data is exactly the same as the original data sent. Cryptographic methods like hash functions and digital signatures are often used to maintain data integrity.

Non-repudiation: Non-repudiation prevents parties from denying their actions. It ensures that a sender cannot deny sending a message, and a recipient cannot deny receiving it. Cryptographic techniques such as digital signatures provide proof of origin, integrity, and receipt of data, holding each party accountable for their actions.

What is Caesar Cipher ?

Caesar cipher is a type of substitution cipher in which each letter of the alphabet is swap by a letter a certain distance away from that letter.

Algorithm

  1. Step 0: Map each letter of the alphabet to its corresponding numerical position (i.e., A = 1, B = 2, ..., Z = 26) for mathematical representation.

What is RSA Cryptography?

RSA (Rivest‚ÄìShamir‚ÄìAdleman) is a public-key cryptosystem and one of the oldest, most reliable methods for securing data transmission. Developed in 1977, it allows data to be encrypted with one key (public) and decrypted with another (private). Its strength relies on the computational difficulty of factoring large prime numbers.

How Does RSA Verify Identity?

Encrypting with the public key and decrypting with the private key is a core feature of RSA, enabling secure communication over untrusted networks. To send sensitive data, the recipient shares their public key with the sender. The sender uses this public key to encrypt the message and transmits it. Only the recipient, who holds the matching private key, can decrypt and read the message. Even if the data is intercepted in transit, it remains unreadable without the private key ‚Äî ensuring confidentiality and secure delivery.

How is RSA Identity Verification?

RSA Identity Verification:RSA can also be used to verify identity by encrypting a message with the sender‚Äôs private key. The recipient uses the sender‚Äôs public key to decrypt it ‚Äî if successful, it proves the message came from the sender and wasn‚Äôt tampered with. This process supports digital signatures. RSA‚Äôs strength lies in the difficulty of factoring large numbers made from multiplying two large primes. Modern RSA keys are typically 2048 bits long, offering strong security, while 1024-bit keys are becoming outdated.

Functionality of an RSA Cryptosystem

RSA works using a pair of keys: a public key (shared with everyone) and a private key (kept secret). These keys are mathematically linked and enable several essential security features:

  • Confidentiality: Ensures only the intended recipient can read the message. The sender encrypts the data with the recipient‚Äôs public key, and only the recipient can decrypt it using their private key.
  • Authentication: Proves the identity of the sender. The sender encrypts data (or a digital signature) with their private key, and anyone with the sender‚Äôs public key can verify its origin.
  • Integrity: Ensures that the message hasn‚Äôt been tampered with during transmission. RSA can be used with digital signatures to verify that the message remains unchanged.
  • Non-repudiation: Prevents senders from denying that they sent a message. Once a message is signed using a private key, it can be verified by others using the public key.
What is RSA Key generation ?

In cryptography, key generation is the process of creating one or more keys to be used for encryption and decryption of data. The system generates two mathematically linked keys: a public key, which is shared openly, and a private key, which is kept secret. These keys are based on large prime numbers and mathematical functions, ensuring that data encrypted with one key can only be decrypted with the other.

Key Generation Algorithm

  1. Step 1: Choose two distinct prime numbers p and q.
    Example: p = 3, q = 11
  2. Step 2: Compute n = p √ó q, which will be used as the modulus for both keys.
    Example: n = 3 √ó 11 = 33
  3. Step 3: Compute Euler‚Äôs Totient function:
    Œ¶(n) = (p ‚àí 1) √ó (q ‚àí 1)
    Example: Œ¶(33) = (3‚àí1)(11‚àí1) = 2 √ó 10 = 20
  4. Step 4: Choose a public exponent e such that:
    • 1 < e < Œ¶(n)
    • gcd(e, Œ¶(n)) = 1 (i.e., e is co-prime with Œ¶(n))
    Example: e = 3
  5. Step 5: Calculate the private exponent d such that:
    (d √ó e) ‚â° 1 mod Œ¶(n)
    This is done using the Extended Euclidean Algorithm.
    Example: d = 7 since 3 √ó 7 = 21 ‚â° 1 mod 20
  6. Step 6: Construct the keys:

What is RSA Encryption?

Once RSA keys are generated, the sender uses the recipient‚Äôs public key to encrypt the original message. This process transforms readable data (plaintext) into an unreadable format called cipher text, which can safely be transmitted over insecure channels such as the internet.

RSA Encryption Algorithm
  1. Step 1: Convert the message (M) into a number using ASCII or other encoding schemes.
  2. Step 2: Use the public key (e, n) to encrypt the message using the formula:
    C = Me mod n
RSA Encryption Example

Public Key: e = 3, n = 33
Message (M): 4

Encryption Formula:
C = Me mod n = 43 mod 33 = 64 mod 33 = 31

What is RSA Decryption?

RSA Decryption is the process of converting encrypted data (cipher text) back into its original readable form (plain text) using the private key. Once the recipient receives the cipher text, they apply their private key to decrypt it, ensuring that only the rightful owner‚Äîwho holds the private key‚Äîcan access the original message.

RSA Decryption Algorithm

  1. Step 1: Take the cipher text C that needs to be decrypted.
  2. Step 2: Use the private key (d, n).
  3. Step 3: Apply the formula:
    M = Cd mod n, where M is the original message.

Example:

  • Cipher Text (C): 31
  • Private Key: (d = 7, n = 33)
  • Decryption Formula: M = 317 mod 33
  • Result: M = 4

üîì Decrypted Message: M = 4
